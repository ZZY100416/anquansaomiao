# 统一安全扫描平台产品介绍

## 1. 产品愿景

- **痛点定位**：传统安全扫描工具分散、结果无法联动、运行时缺失；DevSecOps 流程中安全团队与研发团队协同效率低。
- **解决方案**：构建一体化的“智控安防统一安全扫描平台”（USS），覆盖 SAST/SCA/容器扫描/RASP 运行时防护，实现“扫描前置 + 运行时监控”的闭环。
- **目标用户**：安全工程师、开发团队、运维/DevOps、人事管理层。

## 2. 核心能力矩阵

| 功能域 | 子能力 | 亮点 |
| ------ | ------ | ---- |
| 项目管理 | 项目信息登记、源码/压缩包上传、类型检测 | 上传后自动识别语言栈，落地于 `uploaded_files/`，前端实时显示状态 |
| 扫描编排 | SAST、SCA、容器、RASP | 统一任务列表；支持 JSON 配置自定义参数，例如容器镜像名称、RASP app_id |
| 结果分析 | 漏洞详情、趋势看板、报告导出 | 分级展示风险、提供修复建议；后续支持 PDF/HTML |
| 运行时联动 | OpenRASP 事件聚合 | 自动登录、拉取攻击事件；涵盖 Path Traversal 等实际攻击场景 |
| DevSecOps 集成 | GitHub Actions、Dependabot、API | 支持自动触发扫描、依赖升级建议、REST 接口调用 |

## 3. 技术架构

```
Frontend (React + Ant Design) ──▶ Backend (Flask + SQLAlchemy) ──▶ PostgreSQL
          │                                │
          ▼                                ▼
     JWT Auth                      Scanner Service
                                         │
                                         ▼
       Semgrep / Dependency-Check / pip-audit / npm audit / Trivy / OpenRASP API
```

- Docker Compose 快速部署：`frontend`、`backend`、`postgres`、`redis`
- 扫描结果与上传文件使用 Docker volume 持久化

## 4. 产品优势

1. **一站式覆盖**：静态、依赖、容器、运行时；避免工具孤岛。
2. **轻量部署**：Docker Compose 一键启动，适合离线或内网环境。
3. **排障友好**：沉淀大量 Markdown 手册，对常见故障给出脚本级解决方案。
4. **运行时闭环**：OpenRASP Agent 手工集成方案已验证可行，后台可实时查看攻击事件。
5. **扩展可塑**：Scanner Service 模块化，可插入自研规则或第三方 API。

## 5. 典型场景

1. **代码扫描**：开发提交代码 → CI 触发 USS → SAST/SCA 结果写入 PR 注释。
2. **镜像检测**：上线前容器镜像填入扫描配置 → Trivy 输出 CVE 列表 → DevOps 决策阻断。
3. **运行时告警**：生产环境部署 RASP → USS 定时拉取攻击事件 → 安全值班人员快速复盘。
