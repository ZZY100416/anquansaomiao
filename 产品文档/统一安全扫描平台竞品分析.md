# 统一安全扫描平台竞品分析

> 本文聚焦 SAST/SCA/容器扫描/RASP 综合平台领域，比较自研 USS 与主流商业/开源方案之间的差异，为产品定位和迭代提供决策参考。

## 1. 对比维度

- 能力覆盖：静态、依赖、容器、安全运行时
- 部署模式：SaaS、私有化、本地化
- 成本与授权：开源/商业/订阅
- 可扩展性：规则库、插件、API 集成
- DevSecOps 支持：CI/CD、自动修复、通知

## 2. 竞品概览

| 产品 | 供应商 | 主要功能 | 部署模式 |
| ---- | ------ | -------- | -------- |
| **USS（自研）** | 内部 | SAST、SCA、容器扫描、OpenRASP、CI 集成 | Docker Compose，本地化 |
| Snyk | Snyk Ltd. | SAST、SCA、容器、IaC、修复建议 | SaaS，企业版支持私有部署 |
| Checkmarx One | Checkmarx | SAST、SCA、IAST、Codebashing | SaaS/私有化 |
| Aqua Security | Aqua | 容器、云原生、供应链安全 | SaaS/私有化 |
| 阿里云云安全中心 | 阿里云 | 主机/容器/镜像扫描、合规 | 云服务 |

## 3. 能力对比

| 能力 | USS | Snyk | Checkmarx One | Aqua Security |
| ---- | --- | ---- | -------------- | ------------- |
| **SAST** | Semgrep（Python/Node/Java 等），支持自定义规则 | 多语言，IDE 插件 | 商业引擎，深度规则库 | 主要关注容器，SAST 较弱 |
| **SCA** | Dependency-Check、pip-audit、npm audit | 支持多语言依赖，实时数据库 | 商业数据库 | 强调开源依赖风险 |
| **容器扫描** | Trivy、Hadolint | 支持镜像、registry | 需额外组件 | 强项，含 runtime 保护 |
| **RASP/运行时** | OpenRASP 集成，攻防闭环 | 无 | IAST 可选 | 提供 runtime defense |
| **CI 集成** | REST API、GitHub Actions 模板 | 多种 CI 插件 | 官方集成丰富 | 主要面向企业 devops |
| **报告与指标** | JSON 导出、仪表盘 | 丰富仪表盘、治理建议 | 商业报告 | 企业治理视角 |

## 4. 优势与不足

### 4.1 USS 优势

- 高度本地化：灵活部署，适合安全隔离环境。
- 成本可控：基于开源工具，自研平台层。
- RASP 联动：运行时检测与扫描结果同平台呈现。
- 排障手册完整：涵盖 Dependency-Check、OpenRASP、MySQL 等案例。

### 4.2 USS 不足

- 规则库与漏洞数据库更新需要人工维护。
- 报告形式较原始，缺少 PDF/HTML、邮件通知。
- 缺少商业级别支持，如 SLA、合规认证。

## 5. 机会与策略

1. **差异化**：强化运行时与扫描结果关联，形成进攻链（Kill Chain）视图。
2. **生态开放**：提供插件化接口，允许接入自研扫描或第三方 API。
3. **商业化潜力**：面向中小企业提供离线安全扫描包，按项目计费。
4. **自动修复**：引入 AI/规则推荐，简化修复闭环。

## 6. 结论

- USS 在本地部署、运行时联动方面具备独特优势，适合内部安全平台建设。
- 对比国际竞品，可持续迭代报告、治理能力与自动化通知。
- 建议后续结合组织需求，围绕“本地化 + 运行时闭环 + DevSecOps 集成”方向深耕。
