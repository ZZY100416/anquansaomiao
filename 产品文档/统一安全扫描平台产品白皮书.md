# 统一安全扫描平台产品白皮书

## 1. 前言

随着企业数字化转型的加速，安全风险呈现“左移 + 右护”同时进行的趋势：开发阶段需要快速发现问题，运行阶段要求实时拦截。智控安防统一安全扫描平台（Unified Security Scanner，简称 USS）即是在此背景下诞生的内部安全中台，用于支撑团队 DevSecOps 安全建设与合规落地。

## 2. 产品定位

- **核心定位**：面向研发团队的“扫描 + 运行时”一体化安全平台。
- **价值主张**：
  1. 将 SAST/SCA/容器扫描集中管理，降低工具分散导致的运营成本。
  2. 打通 OpenRASP 运行时防护，实现风险的快速感知和闭环治理。
  3. 支撑本地化部署，满足对外网隔离、数据合规的场景需求。

## 3. 目标用户及痛点

| 角色 | 主要痛点 | USS 对应能力 |
| ---- | -------- | ------------ |
| 安全工程师 | 多工具运维复杂、数据难汇总 | 单平台管理全部扫描任务、集中报表 |
| 开发团队 | 自助扫描困难、修复信息缺失 | 自助创建任务、查看漏洞详情与建议 |
| DevOps/运维 | CI/CD 中安全能力缺位 | REST API、GitHub Actions 集成、自动化脚本 |
| 管理层 | 缺少安全量化指标 | 仪表盘、风险趋势、运行时事件报表 |

## 4. 产品能力全景

1. **扫描编排**：SAST（Semgrep）、SCA（Dependency-Check、pip-audit、npm audit）、容器扫描（Trivy/Hadolint）。
2. **运行时防护联动**：OpenRASP Agent 集成，后台可查看拦截与记录事件。
3. **DevSecOps 集成**：GitHub Actions 模板、Dependabot 协同、API 接口。
4. **运维工具包**：大量排障 Markdown 文档覆盖 Dependency-Check、OpenRASP、MySQL、Docker。
5. **报表与仪表盘**：展示项目数量、漏洞分布、任务趋势，支持 JSON 导出。

## 5. 竞争优势

1. **本地化与可控性**：基于 Docker Compose，部署和升级简单，适用于内网环境。
2. **运行时闭环**：同平台展示扫描结果与实时攻击事件，便于统一响应。
3. **成本优势**：依托开源工具，自研平台层，避免高昂商业授权费用。
4. **知识沉淀**：排障手册覆盖全部场景，降低学习成本，保障项目快速上线。

## 6. 商业模式与应用场景

- **内部模式**：作为集团 DevSecOps 中台，为多个业务线提供安全服务；成本来自基础运维投入。
- **外部模式（规划）**：
  - 提供“离线安全扫描包”，按项目/扫描次数计费。
  - 提供定制化实施支持、规则库维护、运行时部署咨询。
- **典型行业场景**：金融、政企、工业互联网等对数据安全及本地化部署有严格要求的部门。

## 7. 当前成绩（2025-11）

- 平台 Beta 部署完成，覆盖 SAST/SCA/容器/RASP。
- 成功接入 `chatbotAi` 项目，OpenRASP 捕获并拦截路径遍历攻击。
- 形成 40+ 份运维与排障文档，为后续推广提供知识库。

## 8. 发展路线

| 阶段 | 时间 | 核心任务 |
| ---- | ---- | -------- |
| V1.0 (当前) | 2025 Q4 | 平台 MVP、工具整合、RASP 联动、排障手册完成 |
| V1.1 | 2026 Q1 | 报表升级、通知机制、规则库自动更新、RASP 日志配置化 |
| V1.2 | 2026 Q2 | 多租户支持、SaaS 化试点、运营仪表盘与 SLA |
| V2.0 | 规划 | 引入 AI 辅助修复、漏洞治理工作流、外部客户化服务 |

## 9. 风险与应对

| 风险 | 描述 | 应对策略 |
| ---- | ---- | -------- |
| 规则库更新滞后 | 依赖开源社区更新 | 计划建立自动同步脚本 + 安全团队定期 review |
| 运行时数据缺失 | Agent 配置复杂、日志默认关闭 | 发布标准化安装脚本、在平台界面增加检查项 |
| 报告能力不足 | 目前仅支持 JSON | 优先实现 PDF/HTML 导出、告警通知 |
| 资源压力 | 并发扫描导致 CPU/内存飙升 | 引入队列限流、资源度量、扩容方案 |

## 10. 总结

统一安全扫描平台是组织安全体系的重要基础设施，兼顾开发阶段与运行阶段。通过持续迭代和知识沉淀，平台已经验证：

- 可以在本地化环境中快速部署与扩展；
- 能够显著降低多工具维护成本；
- 帮助团队建立安全工作闭环。

下一阶段，将在报表、策略、生态等方向持续发力，使其成为企业安全建设的核心支撑。
